동작 규칙(제안) — 썸 드래그 시 5% 스텝 스냅(기존 슬라이더와 동일 정밀도), Warning은 Critical−5%를 넘지 못하고 그 반대도 같음(교차 불가), 구간 색은 팝오버/메뉴바와 같은 토큰(accentColor/warnColor/critColor) 재사용으로 앱 전체 색 의미 일관 유지.

구현 코스트 정직 고지 — AppKit NSSlider는 썸 2개를 지원하지 않아 커스텀 NSView(마우스 트래킹 + 드로잉 + 접근성)를 새로 만들어야 합니다. 기존 슬라이더 2개 재사용 대비 구현·테스트 부담이 확실히 큽니다(대략 반나절~하루). 접근성(VoiceOver로 각 썸 조작)도 커스텀 구현 몫입니다.
